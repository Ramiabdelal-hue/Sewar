# 🧪 دليل اختبار نظام تتبع Screenshots

## 📋 قائمة الاختبار

### ✅ الاختبار 1: تسجيل محاولة Screenshot

**الخطوات:**
1. افتح الموقع: `https://sewar-1q112.vercel.app`
2. سجل دخول كمشترك
3. اذهب إلى أي صفحة محمية (مثل `/lessons/view`)
4. اضغط على زر **Print Screen**
5. يجب أن ترى شاشة سوداء مع رسالة "محتوى محمي"

**النتيجة المتوقعة:**
- ✅ ظهور overlay أسود
- ✅ تسجيل المحاولة في قاعدة البيانات
- ✅ حفظ: البريد، التاريخ، الصفحة، IP

---

### ✅ الاختبار 2: عرض عدد المحاولات في لوحة الأدمن

**الخطوات:**
1. افتح `/admin/subscribers`
2. سجل دخول كأدمن
3. ابحث عن المشترك الذي اختبرت به
4. انظر إلى عمود "📸 Screenshots"

**النتيجة المتوقعة:**
- ✅ ظهور زر برتقالي مع عدد المحاولات
- ✅ إذا كانت محاولة واحدة: `📸 1`
- ✅ الزر قابل للنقر

---

### ✅ الاختبار 3: عرض تفاصيل المحاولات

**الخطوات:**
1. في صفحة `/admin/subscribers`
2. انقر على زر عدد المحاولات
3. يجب أن تظهر نافذة منبثقة

**النتيجة المتوقعة:**
- ✅ ظهور نافذة بيضاء في المنتصف
- ✅ عرض معلومات المشترك (الاسم، البريد، الهاتف، الاشتراك)
- ✅ قائمة بكل المحاولات مع:
  - التاريخ والوقت بالتفصيل
  - الصفحة
  - عنوان IP
- ✅ زر "إغلاق" في الأسفل

---

### ✅ الاختبار 4: التحذير عند تجاوز 3 محاولات

**الخطوات:**
1. سجل دخول كمشترك
2. حاول أخذ screenshot **4 مرات** في صفحات مختلفة
3. افتح `/admin/subscribers` كأدمن

**النتيجة المتوقعة:**
- ✅ ظهور **بانر أحمر نابض** في أعلى الصفحة
- ✅ رسالة: "يوجد 1 مشترك حاولوا أخذ أكثر من 3 لقطات شاشة"
- ✅ عرض معلومات المشترك في البانر:
  - الاسم
  - البريد الإلكتروني
  - رقم الهاتف
  - عدد المحاولات
- ✅ زر المحاولات في الجدول يصبح **أحمر نابض** مع علامة ⚠️

---

### ✅ الاختبار 5: تمييز المحاولات المشبوهة

**الخطوات:**
1. افتح تفاصيل مشترك لديه أكثر من 3 محاولات
2. انظر إلى قائمة المحاولات

**النتيجة المتوقعة:**
- ✅ المحاولات 1-3: خلفية برتقالية
- ✅ المحاولات 4+: خلفية حمراء مع علامة "⚠️ مشبوه"

---

### ✅ الاختبار 6: عرض التاريخ بالتفصيل

**الخطوات:**
1. افتح تفاصيل أي مشترك لديه محاولات
2. انظر إلى تاريخ كل محاولة

**النتيجة المتوقعة:**
- ✅ التاريخ معروض بالعربية
- ✅ يحتوي على: السنة، الشهر، اليوم، الساعة، الدقيقة، الثانية
- ✅ مثال: "22 أبريل 2026، 14:30:45"

---

### ✅ الاختبار 7: عرض الصفحة و IP

**الخطوات:**
1. افتح تفاصيل أي مشترك
2. انظر إلى تفاصيل كل محاولة

**النتيجة المتوقعة:**
- ✅ عرض الصفحة مع أيقونة 🌐
- ✅ عرض IP مع أيقونة 🔍
- ✅ مثال:
  ```
  🌐 /lessons/view
  🔍 IP: 192.168.1.100
  ```

## 🐛 الأخطاء المحتملة وحلولها

### ❌ المشكلة: لا يتم تسجيل المحاولات

**الأسباب المحتملة:**
1. `localStorage.getItem("userEmail")` فارغ
2. API `/api/activity` لا يعمل
3. مشكلة في قاعدة البيانات

**الحل:**
```javascript
// تحقق من console في المتصفح
console.log(localStorage.getItem("userEmail"));

// تحقق من Network tab
// يجب أن ترى POST request إلى /api/activity
```

---

### ❌ المشكلة: لا يظهر عدد المحاولات في الجدول

**الأسباب المحتملة:**
1. API لا يرجع `screenshotDetails`
2. مشكلة في الـ query

**الحل:**
```javascript
// افتح Network tab
// انظر إلى response من /api/admin/subscribers
// يجب أن يحتوي على:
{
  "data": {
    "subscriptions": [
      {
        "screenshotDetails": {
          "count": 2,
          "attempts": [...]
        }
      }
    ]
  }
}
```

---

### ❌ المشكلة: البانر التحذيري لا يظهر

**الأسباب المحتملة:**
1. عدد المحاولات أقل من 3
2. `warnings` لا يتم إرجاعه من API

**الحل:**
```javascript
// تحقق من response
// يجب أن يحتوي على:
{
  "data": {
    "warnings": {
      "suspiciousScreenshots": 1,
      "suspiciousUsers": [...]
    }
  }
}
```
